# 酒店周边竞品房价监控 + 临时热点活动采集系统（可执行版 v2）

## 0. 文档定位

- 本文档为研发执行基线，目标是让产品、前后端、扩展端、测试、运维在同一口径下协同开发。
- 本文档不替代原始方案说明，优先用于“立项后到上线前”的工程落地。
- 原则：统一字段、统一接口、统一边界、统一验收。

---

## 1. 目标与范围（MVP）

### 1.1 业务目标

- 帮助酒店运营在每日经营中快速感知竞品价格波动与周边活动影响。
- 提供可解释的告警与建议，不做自动调价决策。

### 1.2 MVP范围（强约束）

- 仅支持 `meituan` 平台数据采集（通过 Chrome Extension 在用户已登录后台页面采集）。
- 活动采集仅覆盖：考试类 + 展会类公开数据源。
- 通知渠道仅支持微信（Server酱/WxPusher 二选一接入）。
- AI报告为 P1（可延后上线），不阻塞 P0 主流程验收。

### 1.3 非目标（明确不做）

- 不代登录、不存美团账号密码。
- 不接入 PMS/OTA 自动改价。
- 不承诺“绝对实时”，仅承诺“近实时”。

---

## 2. 统一术语与SLA

### 2.1 统一术语

- **近实时（Near Real-time）**：指数据在 Extension 成功上报后，系统在目标延迟内完成入库、规则计算与推送。
- **数据新鲜度**：最近一次有效采集距当前的时间差。
- **数据过期**：超过阈值未采集（默认24小时）。

### 2.2 SLA（MVP）

- Extension 单次采集+上报：P95 < 10s（网络正常）。
- 上报到告警触达（微信）：P95 < 60s。
- 仪表盘接口 `GET /api/v1/dashboard/price-overview`：P95 < 3s。
- 核心服务月可用性：>= 99.5%。

### 2.3 SLA前提

- 用户需在采集窗口内实际打开美团商家后台对应页面。
- 第三方推送渠道可用且未触发频控。

## 3. 统一数据模型（字段最终版）

> 说明：以下为“实现口径”，命名冲突已收敛。

### 3.1 users

- `id` UUID PK
- `username` VARCHAR(50) UNIQUE NOT NULL
- `email` VARCHAR(255) UNIQUE NOT NULL
- `hashed_password` VARCHAR(255) NOT NULL
- `role` VARCHAR(20) NOT NULL DEFAULT `manager`（MVP仅 `admin/manager`）
- `hotel_name` VARCHAR(255) NOT NULL
- `hotel_location` GEOGRAPHY(Point) NOT NULL
- `is_active` BOOLEAN DEFAULT true
- `created_at` TIMESTAMPTZ
- `updated_at` TIMESTAMPTZ

### 3.2 competitor_hotels

- `id` UUID PK
- `user_id` UUID FK -> users.id
- `name` VARCHAR(255) NOT NULL
- `platform` VARCHAR(20) NOT NULL CHECK IN (`meituan`)  // MVP强约束
- `external_id` VARCHAR(255) NOT NULL
- `room_types` JSONB DEFAULT `[]`
- `is_active` BOOLEAN DEFAULT true
- `created_at` TIMESTAMPTZ
- `updated_at` TIMESTAMPTZ
- UNIQUE(`user_id`, `platform`, `external_id`)

### 3.3 price_time_series（TimescaleDB）

- `time` TIMESTAMPTZ NOT NULL
- `competitor_hotel_id` UUID FK
- `room_type` VARCHAR(100) NOT NULL
- `price` DECIMAL(10,2) NOT NULL
- `availability` BOOLEAN
- `data_source` VARCHAR(20) NOT NULL DEFAULT `extension`
- `collected_at` TIMESTAMPTZ NOT NULL
- `metadata` JSONB DEFAULT `{}`
- PK(`time`, `competitor_hotel_id`, `room_type`)

### 3.4 surrounding_activities

- `id` UUID PK
- `title` VARCHAR(500) NOT NULL
- `description` TEXT
- `start_time` TIMESTAMPTZ NOT NULL
- `end_time` TIMESTAMPTZ NOT NULL
- `location` GEOGRAPHY(Point)
- `address` VARCHAR(500)
- `source` VARCHAR(100) NOT NULL
- `source_url` VARCHAR(500)
- `activity_type` VARCHAR(50) NOT NULL CHECK IN (`exam`, `exhibition`, `other`)
- `estimated_attendance` INTEGER
- `demand_level` VARCHAR(20) NOT NULL CHECK IN (`HIGH`, `MEDIUM`, `LOW`)  // 统一字段名
- `demand_score` DECIMAL(3,2)
- `external_id` VARCHAR(255)
- `collected_at` TIMESTAMPTZ
- UNIQUE(`source`, `external_id`)

### 3.5 extension_devices

- `id` UUID PK
- `user_id` UUID FK
- `device_id` VARCHAR(255) NOT NULL
- `extension_token_hash` VARCHAR(255) NOT NULL
- `status` VARCHAR(20) NOT NULL DEFAULT `offline`
- `last_active_at` TIMESTAMPTZ
- `last_collect_at` TIMESTAMPTZ
- `version` VARCHAR(20)
- `total_reports` INTEGER DEFAULT 0
- `created_at` TIMESTAMPTZ
- UNIQUE(`user_id`, `device_id`)

### 3.6 alert_rules / alert_records / notifications / ai_reports

- 结构沿原方案，新增执行要求：
  - 所有业务表必须包含 `created_at`。
  - 所有用户可见查询必须强制按 `user_id` 过滤。
  - 消息发送状态必须可追踪：`pending/sent/failed`。

## 4. API协议（最终版）

> 统一响应包：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 4.1 认证

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`

约束：
- 注册默认角色为 `manager`（与DB一致）。

### 4.2 Extension

- `POST /api/v1/extension/register`（JWT）
- `POST /api/v1/extension/report`（`X-Extension-Token`）
- `GET /api/v1/extension/status`（JWT）

#### `POST /api/v1/extension/report` 请求体（统一版）

```json
{
  "type": "competitor|business|benchmark",
  "source": "meituan_merchant",
  "data": {
    "competitors": [
      {
        "name": "竞品酒店A",
        "room_type": "豪华大床房",
        "price": 480.0,
        "availability": true,
        "rating": "4.5",
        "sales": "120"
      }
    ],
    "business": {},
    "benchmark": {}
  },
  "url": "https://e.meituan.com/...",
  "captured_at": "2026-04-08T10:00:00Z"
}
```

约束：
- `data` 固定为对象，`competitors` 固定为数组（禁止直接上传裸数组）。
- 后端必须做 schema 校验，失败返回 `422`。

### 4.3 核心业务

- `GET/POST/PUT/DELETE /api/v1/competitors`
- `GET /api/v1/competitors/{id}/price-history`
- `GET /api/v1/dashboard/price-overview`
- `GET /api/v1/activities/calendar`
- `POST /api/v1/alert-rules`
- `GET /api/v1/notifications`

### 4.4 AI与任务（P1）

- `POST /api/v1/ai/generate-weekly-report`
- `POST /api/v1/tasks/crawl/activity`

## 5. Chrome Extension执行规范（MV3）

### 5.1 关键约束

- 仅申请最小权限：`storage` + 必要 `host_permissions`。
- Token 仅存本地，不写日志明文。
- 上报失败进入本地重试队列，指数退避，最大重试次数可配置。

### 5.2 采集触发策略

- 页面初次加载延时采集（默认2s）。
- SPA路由切换监听并去抖（默认2s窗口）。
- 同页面5分钟内同类型数据重复采集需去重。

### 5.3 选择器配置拉取（修正版）

```javascript
async function getSelectors() {
  const { meituan_selectors } = await chrome.storage.local.get("meituan_selectors");
  const { selectors_updated_at } = await chrome.storage.local.get("selectors_updated_at");
  const ONE_HOUR = 60 * 60 * 1000;
  if (!meituan_selectors || !selectors_updated_at || Date.now() - selectors_updated_at > ONE_HOUR) {
    const remote = await fetchSelectorsFromBackend();
    await chrome.storage.local.set({
      meituan_selectors: remote,
      selectors_updated_at: Date.now()
    });
    return remote;
  }
  return meituan_selectors;
}
```

## 6. 后端执行规范

### 6.1 技术栈

- FastAPI + SQLAlchemy + Alembic
- Celery + Redis
- PostgreSQL + TimescaleDB + PostGIS

### 6.2 关键实现要求

- `extension/report` 必须异步触发告警检测任务，不阻塞主请求。
- 告警引擎默认规则窗口：1h，可配置。
- 所有外部调用（微信/模型）统一做超时、重试、熔断与降级。
- 所有关键任务写 `task_log` 并可按 `task_id` 查询。

### 6.3 安全与隔离

- JWT + RBAC；Extension使用独立 Token 认证。
- `extension_token` 仅存 hash。
- 接口审计日志至少保留90天（MVP可按存储成本调至30天）。

## 8. 任务编排与调度

### 8.1 周期任务（Celery Beat）

- 活动采集：每日定时（默认 06:00 / 14:00 / 22:00）。
- Extension离线检查：每15分钟。
- 报告生成（P1）：每日/每周定时任务。

### 8.2 事件任务

- Extension上报 -> 入库成功 -> 告警检测 -> 命中则通知推送。

### 8.3 幂等与去重

- 活动数据以 `source + external_id` 幂等。
- 房价时序以 `(time, competitor_hotel_id, room_type)` 幂等写入。

## 9. 监控与告警

### 9.1 技术监控

- API P95延迟、5xx错误率
- Celery队列积压长度
- DB慢查询与连接池占用
- 第三方推送失败率

### 9.2 业务监控

- Extension活跃率
- 上报成功率
- 告警触发延迟
- 活动采集成功率

### 9.3 告警阈值（默认）

- 5xx > 2% 持续5分钟
- 告警触发延迟 P95 > 120s 持续10分钟
- Extension日活下降 > 30%
